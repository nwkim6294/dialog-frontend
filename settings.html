🎨 챗봇 화면 구성 & 버튼 설명

⏱️

히스토리 버튼

  • 이전 대화 기록 확인
  • 과거 질문/답변 재확인
  • 원하는 대화 클릭하여 이동
➕

새 대화 버튼

  • 완전히 새로운 대화 시작
  • 이전 컨텍스트 지우고 처음부터
  • 다른 주제로 질문할 때 사용
✕

닫기 버튼

  • 챗봇 창 닫기
  • 대화 내용 자동 저장
  • 다시 열면 마지막 대화 이어서 가능

💡 언제 새 대화를 시작할까요?

• 회의록 검색 → IT 용어 검색으로 주제를 바꿀 때

• 챗봇이 엉뚱한 답변을 계속할 때 (컨텍스트 리셋)

• 완전히 다른 회의에 대해 물어볼 때

🔍 회의록 검색 완벽 가이드

📅 기간으로 찾기
  • "11월 회의 뭐있어?"
  • "지난주 회의 보여줘"
  • "10월 1일부터 10월 15일까지"
  • "최근 회의 뭐있어?"
🔑 키워드로 찾기
  • "API 관련 회의 뭐있어?"
  • "배포 관련 논의한 회의"
  • "DB 설계 회의 찾아줘"
  • "스프린트 회의 보여줘"
👤 사람으로 찾기
  • "장문선이 참석한 회의"
  • "장문선 할일 뭐있어?"
  • "킥오프 회의 참석자 알려줘"
✅ 내 할일 확인
  • "내가 해야 할 일 뭐야?"
  • "완료한 일 보여줘"
  • "남은 할일 뭐있어?"

💡 똑똑한 검색 팁

키워드 + 기간을 함께 쓰면 더 정확해요! "지난주 API 회의"처럼요.

원하는 결과가 안 나오면 챗봇이 자동으로 범위를 넓혀서 다시 찾아줘요.

📖 IT 용어사전 활용법

💻 개발 용어
  • "REST API가 뭐야?"
  • "GraphQL이 뭐야?"
  • "컨테이너화가 뭐야?"
🗄️ 데이터베이스 용어
  • "정규화가 뭐야?"
  • "인덱스 설명해줘"
  • "트랜잭션이 뭐야?"
🔐 보안 용어
  • "JWT가 뭐야?"
  • "XSS 공격이 뭐야?"
  • "CORS 설명해줘"
🔤 약어/줄임말
  • "CI/CD가 뭐야?"
  • "PM이 뭐야?"
  • "MSA가 뭐야?"

✨ 용어사전 특징

• 쉬운 말로 설명해요 (전문 용어 최소화)

• 약어는 풀어서 알려줘요 (PM → Project Manager)

• 오타가 있어도 이해해요 ("정규와" → "정규화")

🔥 챗봇 활용 꿀팁

💬 대화 이어가기 (컨텍스트 유지)

회의를 한 번 선택하면, 그 다음 질문부터는 "이 회의"를 자동으로 기억해요!

✅ 이렇게 물어보세요
"API 회의 찾아줘"
→ "1번"
→ "내 할일 뭐야?"
→ "다른 사람은?"
→ "예산 얼마였어?"
회의 선택 후 계속 질문 가능!
❌ 이렇게 하지 마세요
"API 회의 찾아줘"
→ "1번"
→ "API 게이트웨이 회의에서 내 할일 뭐야?"
(불필요하게 회의명 반복)
이미 선택했으니 반복 안 해도 돼요

🎯 한 번에 여러 질문하기

여러 정보가 궁금할 땐 한 번에 물어보세요!

  • "11월 스프린트 회의에서 예산이랑 일정 알려줘"
  • "DB 마이그레이션 회의 참석자랑 내 할일 보여줘"
  • "지난주 회의들 제목이랑 날짜만 간단히 알려줘"

🔢 번호로 빠르게 선택하기

회의 목록이나 할일 목록에서 번호로 선택할 수 있어요!

"11월 회의 뭐있어?"
📋 11월 회의 3건
1️⃣ 마케팅 전략 회의
2️⃣ 스프린트 계획 회의
3️⃣ DB 설계 회의
"2"
📋 스프린트 계획 회의 상세 정보 표시

💡 번호 선택 꿀팁

• 단순히 "1", "2", "3" 만 입력해도 인식!

• "완료 1", "예정 2" 같은 상태 + 번호도 가능

• 여러 개 선택: "1, 3, 5" (쉼표로 구분)

🎭 주제 바뀔 땐 "새 대화" 버튼!

챗봇이 이전 대화를 계속 기억해서 엉뚱한 답변을 할 때가 있어요.

✅ 이럴 때 새 대화!
회의록 검색 하다가
→ IT 용어 검색으로 전환
→ 상단 ➕ 버튼 클릭
→ 깨끗하게 새 시작!
컨텍스트 리셋으로 정확한 답변
✅ 이럴 때도!
A 회의 보다가
→ 완전히 다른 B 회의 검색
→ 새 대화 시작
→ 혼동 없이 B 회의 검색
이전 회의 정보 섞이는 거 방지

💬 대화 흐름 정리

  • 같은 회의 계속 질문: 그냥 이어서 물어보기
  • 다른 회의로 전환: 새 대화 버튼 누르기
  • IT 용어 궁금: 새 대화 시작하고 질문
  • 이전 대화 보기: 히스토리 버튼 (⏱️) 클릭

✍️ 효과적인 질문 작성법

✅ 좋은 예시
"지난주 마케팅 회의에서 예산 얼마로 결정했어?"
기간 + 키워드 + 구체적 질문
❌ 아쉬운 예시
"예산"
너무 짧아서 의도 파악 어려움
✅ 좋은 예시
"API 게이트웨이 회의에서 김나운님 할일 뭐였어?"
회의 + 담당자 명시
❌ 아쉬운 예시
"할일"
누구의 할일인지 불명확
✅ 좋은 예시
"10월 배포 회의 누가 참석했어?"
기간 + 회의 주제 + 질문
❌ 아쉬운 예시
"누가"
어떤 회의인지 불명확
✅ 좋은 예시
"API가 뭐야?" 또는 "에이피아이가 뭐야?"
오타/발음 그대로 써도 OK
❌ 아쉬운 예시
"그거"
무엇을 말하는지 불명확

💡 질문 잘하는 3가지 원칙

1. 구체적으로:

"회의"보다 "11월 스프린트 회의"

2. 맥락 제공:

"할일"보다 "API 게이트웨이 회의에서 내 할일"

3. 자연스럽게:

평소 말하듯이 편하게 물어보세요!

💬 이런 것도 가능해요!

  • 여러 조건 조합: "지난주 DB 회의" → 없으면 자동으로 "DB 회의" 전체 검색
  • 상태 필터: "완료한 일", "남은 할일", "예정된 회의"
  • 번호 선택: 회의 목록에서 "1", "2", "완료 1", "예정 2" 같은 번호로 선택
  • 자연어 날짜: "오늘", "어제", "이번주", "지난달", "최근"

🚀 회의 시작 전 준비하기

STEP 1: 기본 정보 입력
필수 입력 항목
  • 회의 제목: 나중에 찾기 쉽도록 명확하게 입력
  • 회의 일시: 실제 회의 시작 시간 선택

좋은 제목 예시:
✅ "11월 마케팅 전략 회의"
✅ "2024 Q4 스프린트 계획"
✅ "API 게이트웨이 설계 논의"

아쉬운 제목:
❌ "회의"
❌ "미팅"

STEP 2: 참석자 추가

회의에 참석하는 사람들의 이름을 미리 등록하세요.

1. 참석자 입력창에 이름 입력
2. "추가" 버튼 클릭
3. 등록된 참석자 목록 확인
4. 잘못 추가했다면 ✕ 버튼으로 삭제

💡 참석자 등록의 장점

• 나중에 "장문선이 참석한 회의" 같은 검색이 가능합니다

• 회의 후 각 참석자별 할 일을 따로 확인할 수 있습니다

STEP 3: 하이라이트 키워드 설정

회의 중 중요하게 다룰 키워드를 미리 등록하면, 해당 단어가 나올 때 하이라이트 됩니다.

✅ 추천 키워드
예산, 일정, 마감일, 담당자, 우선순위, 결정사항
중요한 의사결정 관련 단어
✅ 프로젝트 관련
배포, 테스트, 리뷰, 버그, 이슈
프로젝트 진행 중요 단어
STEP 4: 마이크 테스트

녹음 시작 전 마이크가 정상 작동하는지 꼭 확인하세요!

1. "테스트 시작" 버튼 클릭
🎤 마이크 접근 권한 허용 팝업 → "허용" 클릭
2. 마이크에 대고 말하기
📊 음량 표시줄이 움직이면 정상!

⚠️ 마이크가 안 잡힐 때

• 브라우저 설정에서 마이크 권한 확인

• 다른 프로그램이 마이크를 사용 중인지 확인

• 외장 마이크 사용 시 연결 상태 확인

✅ 모든 준비가 끝났다면

우측 상단의 "회의 시작" 버튼을 누르면 회의 녹음 화면으로 이동합니다!

🎬 회의 중 녹음하기

녹음 시작하기
1. 우측 사이드바의 "녹음 시작" 버튼 클릭
⏺️ 타이머가 시작되고 오디오 바가 움직입니다
2. 회의 진행하기

🎯 녹음 중 알아두면 좋은 것

• 녹음 중에도 "일시정지" 가능 (화장실, 휴식 시간 등)

실시간 대화 기록 보기

왼쪽 화면에서 실시간으로 변환되는 텍스트를 확인할 수 있습니다.

대화 기록 화면 구성
  • 참석자 이름: 회의 참석자 표시
  • 시간 정보: 몇 시 몇 분에 말했는지 표시
  • 하이라이트: 설정한 키워드를 강조
  • 자동 스크롤: 체크하면 최신 대화로 자동 이동

화면 예시:
[10:23] 예산은 총 500만원으로 확정했습니다.

[10:24] 네, 그럼 일정은 언제까지인가요?

설정한 키워드가 대화 중 등장하면, 해당 단어가 회의록에서 자동으로 강조 표시됩니다.

회의록 완성 화면

회의 종료 후 AI가 자동으로 생성하는 내용들:

✅ 자동 생성되는 것들
  • 전체 대화록: 발화자별 전체 대화 내용
  • 회의 목적: 핵심 목적을 명사형으로 한 문장 요약
  • 주요 안건: 논의된 안건 3~5가지를 쉼표로 구분
  • AI 요약: 결정 사항과 향후 계획 위주로 3문장 내외 요약
  • 중요도 판정: 높음/보통/낮음 + 판정 사유
  • 핵심 키워드: 직무 관점에서 중요한 명사 키워드 5개
  • 내 할 일: 본인/지시받은/팀 업무 자동 추출 + 날짜 계산
  • 하이라이트 모음: 키워드가 포함된 발언만 모음
  • 참석자 통계: 각 참석자의 발언 시간 및 비율

중요도 판정 예시:
높음 - 예산 확정 및 주요 일정이 결정됨
보통 - 일반적인 진행 상황 공유 회의
낮음 - 단순 정보 전달 및 확인

🎯 직무별 맞춤 분석 (설정 페이지에서 직무 선택)
  • 기획자(PM): 일정, 리소스, 주요 결정사항 중심

Q. 녹음 중간에 실수로 창을 닫으면 어떻게 되나요?

A. 걱정 마세요! 그 시점까지의 녹음은 자동 저장됩니다.

Q. 여러 사람이 동시에 말하면 어떻게 되나요?

A. 가장 크게 말하는 사람의 음성이 우선 인식됩니다. 정확한 기록을 위해 한 명씩 발언하는 것을 추천합니다.

Q. 회의 중간에 일시정지하면 시간이 기록되나요?

A. 일시정지 구간은 녹음되지 않고, 타이머도 멈춥니다. 재개하면 이어서 녹음됩니다.

Q. 방언이나 사투리도 인식되나요?

A. 표준어보다는 정확도가 낮을 수 있지만, 명확하게 발음하면 대부분 인식됩니다.

시나리오 3: PDF 다운로드 + 캘린더 연동 조합

상황: 중요한 회의가 끝나고 팀원들과 공유하며 일정을 관리합니다.

📥 팀원 공유용
1. 회의록 페이지 우측 상단 "PDF 다운로드"
2. Slack/이메일로 팀원들에게 전송
3. 각자 직무별로 읽고 파악
공식 회의록 배포
📅 내 일정 관리용
1. "내 할일" 목록에서 체크박스 선택
2. "캘린더 연동" 버튼 클릭
3. 구글 캘린더에 자동 등록
개인 업무 트래킹
📊 업무 흐름:
회의 녹음 → AI 분석 → PDF 다운로드 (공유용)
→ 내 할일 추출 → 캘린더 등록 (개인용)
→ 챗봇으로 나중에 빠르게 검색

🎯 효과

• 팀 협업: PDF 공유로 투명성 확보

• 개인 생산성: 캘린더로 마감일 자동 관리

• 장기 기록: 챗봇으로 몇 달 전 회의도 즉시 검색
